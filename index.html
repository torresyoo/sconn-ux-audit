P1UX-01

검색 결과가 없을 때 다음 행동을 안내하지 않고, 안내 문구가 검색 맥락과 맞지 않음

접속 경로홈 → 상단 검색창 → 검색어 입력(결과 0건) → 검색 결과 화면
화면 위치https://sconn.io/search/?st=book&sw=<검색어> (전체 0건)

현재 상태

결과가 0건일 때 “조회 가능한 상품이 없습니다. 선택하신 카테고리 또는 조건을 변경해주세요.”만 표시되고, 다음 행동으로 이어지는 버튼이 없습니다.

문제가 되는 이유

검색창에 책 제목을 입력했는데 안내는 “카테고리 또는 조건을 변경”하라고 말합니다. 화면에는 바꿀 카테고리나 조건이 보이지 않아 무엇을 해야 할지 알기 어렵습니다. 결과가 없을 때가 이탈이 가장 큰 지점인데 회복 경로가 없습니다.

개선 이유

검색 결과 없음 화면은 실패를 알리는 곳이 아니라 다음 행동을 주는 곳이어야 합니다. 푸터에 “도서 신청하기”가 있지만 이 화면에서는 노출되지 않아 정작 필요한 순간에 찾지 못합니다.

개선 방식

문구를 검색 맥락(검색어)에 맞게 바꾸고 검색어를 다시 보여 준 뒤, 회복 버튼 두 개(전체 도서 둘러보기 · 찾는 책 신청하기)를 둡니다. 오타·띄어쓰기 검색 팁을 한 줄 덧붙입니다.

권장 문구

  • “‘{검색어}’에 대한 검색 결과가 없어요.”
  • “오타가 없는지, 띄어쓰기를 바꿔보거나 책 제목 일부만 입력해 보세요.”
  • 버튼: “전체 도서 둘러보기” / “찾는 책 신청하기”

장바구니 빈 화면에 다음 행동이 없고, “?” 아이콘이 빈 상태가 아니라 오류처럼 보임

접속 경로상단 장바구니 아이콘 → 장바구니(담긴 상품 없음)
화면 위치https://sconn.io/cart/ (빈 장바구니)

현재 상태

“장바구니가 비어있습니다.” 한 줄과 물음표(?) 상자 아이콘만 보입니다. 도서를 보러 가는 버튼이 없습니다.

문제가 되는 이유

물음표 아이콘은 ‘비어 있음’보다 ‘알 수 없음/오류’로 읽혀, 정상 상태인데도 잘못된 화면처럼 느껴집니다. 여기서 멈추면 사용자는 직접 메뉴를 찾아 되돌아가야 합니다.

개선 이유

빈 장바구니는 구매 흐름이 끊기는 지점입니다. 다음 행동(둘러보기, 관심 도서)을 한 번의 클릭으로 이어 주면 탐색이 끊기지 않습니다.

개선 방식

“?” 아이콘을 빈 장바구니 아이콘으로 바꾸고, 문구를 다듬은 뒤 도서를 보러 가는 버튼을 둡니다. 이미 있는 관심(찜) 기능과 연결합니다.

  • “관심 있는 교재를 담아 두면 한 번에 결제할 수 있어요.”

P2UX-03

날짜 표기가 화면마다 제각각이고, 상품 상세 출간일은 구분 기호가 없음

접속 경로상품 상세 · 장바구니 · 결제내역 등 날짜가 보이는 화면 전반
화면 위치상품 상세 출간일 “20260415” / 장바구니·결제 대여기간 “~2026/11/30” / 결제내역 구매일 “2026.04.03”

현재 상태

같은 플랫폼 안에서 날짜가 네 가지 형식으로 섞여 있습니다. 상품 상세 출간일은 “20260415”처럼 구분 없는 8자리, 장바구니·체크아웃은 “2026/11/30”, 결제내역은 “2026.04.03”, 개인정보 변경의 생년월일·수신동의일은 “1998-09-01”입니다.

문제가 되는 이유

“20260415”는 사용자가 2026·04·15로 직접 끊어 읽어야 합니다. 화면마다 형식이 달라 같은 ‘날짜’를 매번 다르게 해석해야 하고, 출간일·대여 만료일처럼 판단에 쓰는 정보의 읽는 비용이 커집니다.

개선 이유

결제내역은 이미 “2026.04.03”으로 잘 표기하므로 플랫폼은 날짜를 포맷할 수 있습니다. 한 가지 형식으로 통일하면 다시 해석하지 않고 바로 읽히고, 페이지 수 “140페이지” 표기와도 정돈됩니다.

개선 방식

모든 날짜를 “YYYY.MM.DD”로 통일합니다(예: 2026.04.15, ~2026.11.30까지). 표시 형식만 맞추면 되는 데이터 가공 문제입니다.

권장 문구

  • “출간일 2026.04.15”
  • “대여기간 ~2026.11.30까지”
  • “구매일 2026.04.03”

P2UX-04

상품 상세의 찜(하트) 버튼에 글자 라벨과 툴팁이 없어 기능을 짐작해야 함

접속 경로상품 상세 → 가격 영역 하단 버튼(하트 / 장바구니 추가 / 구매하기)
화면 위치https://sconn.io/product/<상품번호> (구매 버튼 영역)

현재 상태

“장바구니 추가”, “구매하기”에는 글자가 있지만 왼쪽 하트 버튼에는 글자 라벨도 툴팁도 없습니다.

문제가 되는 이유

하트가 찜인지 좋아요인지 아이콘만으로는 단정하기 어렵습니다. 옆 두 버튼은 글자로 동작을 알 수 있어 라벨 없는 하트만 예측이 어렵고, 스크린리더 사용자는 목적을 알 수 없습니다.

개선 이유

교재 스토어에서 찜은 자주 쓰는 기능입니다. 라벨과 접근성 이름을 붙이면 처음 온 사용자도 학습 없이 바로 알아봅니다. 마이페이지에서 부르는 “위시리스트” 용어와도 맞추면 좋습니다.

개선 방식

하트 옆에 “찜” 글자를 붙이고 접근성 이름(aria-label “찜하기”)을 추가합니다. 담은 뒤에는 채워진 하트와 “찜 완료”로 상태를 보여 줍니다.

권장 문구

현재 상태

문제가 되는 이유

도서 표지·배너에 대체 텍스트가 없으면 화면을 못 보는 사용자는 어떤 책인지 알 수 없고, 아이콘만 있는 링크·버튼(장바구니·찜·검색 등)은 무슨 기능인지 음성으로 읽히지 않습니다. 대비가 낮은 보조 텍스트는 저시력 사용자가 읽기 어렵습니다. 대체 텍스트·이름 누락은 WCAG 2 가장 기본(Level A) 항목입니다.

개선 이유

교재는 시각장애·저시력 학습자도 쓰는 콘텐츠라 접근성은 선택이 아니라 기본 품질입니다. 대체 텍스트·접근성 이름·대비만 갖춰도 보조기기 사용자가 같은 흐름으로 탐색·구매할 수 있고, 비장애 사용자의 이해도 함께 좋아집니다.

개선 방식

구독 ‘무료 체험’ 시작 버튼 옆에 자동 결제 전환 조건·해지 방법이 분명하지 않음

현재 상태

문제가 되는 이유

“무료 체험”만 강조되고 결제 전환 조건이 작은 주석에 흩어져 있으면, 사용자는 무료로만 알고 시작했다가 7일 뒤 14,900원이 결제돼 예상치 못한 비용을 겪을 수 있습니다. 비용이 생기는 결정은 결정하는 그 자리에서 분명히 보여야 합니다.

개선 이유

현재 상태

스콘 충전 화면에서 ‘원’과 ‘스콘’ 단위가 섞여 있고, 금액을 고르기 전엔 합계가 비어 있음

현재 상태

잔액을 “보유 원 : 1,361,630스콘”으로, 충전 시 받는 양을 “지급 예정 원 : 10,300원”으로 표기합니다(받는 것은 스콘). 충전 금액을 고르기 전에는 “총 결제 금액 : 원”처럼 숫자가 비어 있습니다.

문제가 되는 이유

‘스콘’은 서비스 이름이자 포인트 단위로 함께 쓰여 헷갈리는데, 라벨은 ‘원’인데 값은 ‘스콘’이고, 받는 스콘을 ‘원’으로 적어 단위가 뒤섞입니다. 결제 직전 화면에서 단위가 어긋나면 얼마를 내고 얼마를 받는지 다시 따져야 합니다. 금액을 고르기 전 “: 원” 빈 표기는 입력이 누락된 듯 보입니다.

개선 이유

충전은 비용이 오가는 단계라 단위가 정확해야 합니다. 결제 금액은 ‘원’, 잔액·적립은 ‘스콘’으로 일관되게 구분하고 1스콘=1원 관계를 한 번 알려 주면 다시 해석하지 않아도 됩니다.

개선 방식

잔액 라벨을 “보유 스콘”으로, 받는 양 라벨을 “받는 스콘 … 스콘”으로 바꿉니다. 금액 미선택 시에는 “총 결제 금액 0원” 또는 “충전 금액을 선택해 주세요”로 안내합니다. 화면 어딘가에 “1스콘 = 1원”을 한 줄로 둡니다.

문제가 되는 이유

같은 대상을 다른 표기로 부르면 사용자는 같은 것인지 잠깐 확인하게 됩니다. 작은 차이라도 화면을 오갈 때 인지 비용이 쌓입니다.

개선 이유

같은 기능은 같은 이름으로 부르는 것이 일관성의 기본입니다. 표기를 통일하면 메뉴와 카드가 같은 곳을 가리킨다는 것이 분명해집니다.

개선 방식

“내 서재”로 띄어쓰기를 통일하고, 찜/위시리스트 중 하나로 용어를 정해 상품 화면과 마이페이지에서 같게 씁니다.

P1UX-08

웹뷰어 주석 도구가 아이콘만 있고 기능명·툴팁·선택 안내가 없어, 무엇을 하는 도구인지 알기 어려움

접속 경로로그인 → 마이페이지 → 내 서재 → 도서 “웹에서 보기” → 웹뷰어(viewer.sconn.io) → 상단 주석 도구
화면 위치viewer.sconn.io — 상단 가운데 주석 도구 막대(형광펜·밑줄·텍스트·펜·지우개) 및 선택 시 나타나는 서식 막대

현재 상태

주석 도구가 글자 라벨 없이 아이콘으로만 제공됩니다. 특히 비슷한 “T” 모양 아이콘 세 개(노란 T, 밑줄 T, Tᴛ)가 나란히 있어 형광펜·밑줄·텍스트가 구분되지 않습니다. 도구를 골라도 “무엇이 켜졌는지, 어떻게 쓰는지” 안내가 없고, 마우스를 올려도 툴팁이 뜨지 않습니다. 전체화면·설정·색상 선택 등 다른 아이콘 버튼도 같습니다.

문제가 되는 이유

“필기 가능한 전자책”은 주석이 핵심 기능인데, 아이콘만으로는 형광펜인지 밑줄인지 텍스트 입력인지 예측하기 어렵습니다. 실제 사용 중인 이용자도 작동 방법을 헷갈렸습니다. 도구를 고른 뒤에도 선택 상태나 사용법(문장을 드래그하라는 등) 안내가 없어, 매번 눌러 보며 익혀야 합니다. 스크린리더 사용자는 도구 목적을 알 수 없습니다.

개선 이유

주석 도구는 반복해서 쓰는 핵심 기능이라, 처음 쓰는 사람도 학습 없이 알아보고 다시 쓰는 사람도 상태를 바로 확인할 수 있어야 합니다. 아이콘에 기능명과 툴팁, 선택 직후 한 줄 안내를 더하면 “눌러 보며 익히는” 부담이 사라집니다.

개선 방식

각 도구 아이콘 아래(또는 옆)에 기능명을 붙이고, 마우스를 올리면 툴팁을 띄웁니다. 도구를 선택하면 “형광펜을 켰어요. 문장을 드래그해 표시하세요”처럼 한 줄 사용 안내를 보여 주고, 선택된 도구는 눌린 상태로 강조합니다. 서로 닮은 “T” 아이콘 세 개는 형태를 더 분명히 구분합니다. 모든 아이콘 버튼에 접근성 이름(aria-label)을 추가합니다.

권장 문구

  • 아이콘 + 라벨: “형광펜 / 밑줄 / 텍스트 / 펜 / 지우개”
  • 툴팁: “형광펜 · 문장을 드래그해 표시”
  • 선택 직후: “형광펜을 켰어요. 문장을 드래그해 표시하세요.”

P2UX-09

개인정보 변경에서 휴대폰 입력 문제를 “다른 브라우저에서 진행하라”는 안내로 떠넘김

접속 경로로그인 → 마이페이지 → 개인 정보 변경 → 휴대폰(필수)
화면 위치https://sconn.io/my-account/edit-account/ (휴대폰 입력 항목)

현재 상태

휴대폰 항목 아래에 “핸드폰 번호 입력이 안 될 경우, 사파리 또는 크롬 앱에서 진행해주세요.”라는 안내가 있습니다.

문제가 되는 이유

이 문구는 입력이 안 되는 상황이 있다는 것을 전제로, 해결을 사용자에게 “브라우저를 바꿔서 다시 하라”고 떠넘깁니다. 사용자는 왜 안 되는지 모른 채 앱·브라우저를 옮겨 다녀야 하고, 필수 항목이라 변경 자체가 막힐 수 있습니다. 임시 안내가 미해결 문제를 덮고 있습니다.

개선 이유

개인정보(특히 본인 인증과 연결되는 휴대폰)는 어느 브라우저에서도 입력돼야 합니다. 안내로 우회시키기보다 입력이 되도록 고치는 것이 맞습니다.

개선 방식

입력이 막히는 원인을 수정해 모든 환경에서 입력·인증이 되게 합니다. 그 전까지도 “사파리/크롬에서 하라”는 회피성 문구 대신, 정상 입력 필드와 인증 흐름(번호 입력 → 인증번호 확인)을 제공하고, 오류가 나면 무엇이 문제인지 구체적으로 안내합니다.

권장 문구

  • (정상) 번호 입력 → “인증번호 받기” → 인증번호 확인
  • (오류 시) “인증 문자를 보내지 못했어요. 번호를 확인하고 다시 시도해 주세요.”
